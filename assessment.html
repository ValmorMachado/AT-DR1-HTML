Design Responsivo

Quando é recomendando a criação de um site separado para mobile? Seria este o caso deste site da Câmara de Deputados?

Ter um site mobile ou um site responsivo é fundamental para uma boa experiência do usuário e um bom ranqueamento do seu endereço.

A versão mobile é preferível em projetos mais complexos que demandam mecanismos específicos na utilização e experiência do usuário. O ideal é que seja feito um estudo com a equipe para identificar se a página na internet exige esse tipo de site.

A maior vantagem do site responsivo é que toda a programação é feita uma só vez em uma única versão, o que consequentemente requer um investimento mais baixo e menos manutenção. Inclusive, a maioria das empresas de desenvolvimento de sites já incluem a responsividade como um requisito básico dos projetos.

Vale lembrar que uma solução não é superior à outra: elas atingem objetivos diferentes de acordo com necessidades específicas. Por isso, deve ser bem estudada quais são as demandas do seu site para fazer a escolha mais acertada.

O site da Câmara dos Deputados apresenta mais informações de textos, com imagens, dispostas em forma de lista. Sem necessidade de criar dois projetos separados. O site responsivo é capaz de atender bem usuários mobile e web.

No caso de um CSS específico para impressão (print), como ele deve ser inserido?

Usamos a tag link para referenciar o arquivo css de impressão e o atributo media=“print” após a página de estilos principal como no exemplo abaixo:

                    
                        <link type="text/css" rel="stylesheet" href="principal.css" />
                        <link type="text/css" rel="stylesheet" media="print" href="imprimir.css" />

No exemplo anterior os estilos dentro de imprimir.css serão aplicados após o estilo principal.

Podemos também mudar no próprio CSS através do @Media Rule. Dessa forma as regras css são sobrescritas na hora da impressão. 

                    
                        /* Exemplo da folha de estilo principal */
                        div {
                        padding-bottom: 2em;
                        color: #FFFFFF;
                        background-color: #000000;
                        }

                        /* Quando imprimir usa as regras abaixo */
                        @media print {

                            div {
                                padding-bottom: 1.5em;
                                color: #000000;
                                background-color: #FFFFFF;
                            }
                        }

Multimídia Web:

O elemento audio permite como elemento-filho o elemento source. Quais são os atributos que o source permite e qual conteúdo eles suportam? Quais os outros atributos do elemento audio e quando devem ser utilizados?

No HTML 5, o elemento source substitui o atributo src.

Atributos source:

  • src: Endereço do arquivo de mídia;
  • type: Tipo de mídia: wav, wave, mp3, ogg;
    • Permite usar o atributo codec.
  • media: São os meios que devem ser apresentados os áudios;
  • preload: Carregamento do aúdio quando é lida a página(auto metadata none);
  • loop: Quando o aúdio termina, volta para o inicio;
  • controls: Customiza os controles a serem mostrados.

O elemento <video/> permite como elemento-filho o elemento source. Quais são os atributos que o source permite e qual conteúdo eles suportam? Quais os outros atributos do elemento video e quando devem ser utilizados?

A TAG vídeo pede obrigatoriamente os atributos src e type.

Formatos de vídeos suportados: webm, mp4 e ogg.

Atributos source:

  • src: Endereço do arquivo de mídia;
  • height: Altura do vídeo;
  • width: Largura do vídeo;
  • autoplay: Executado o vídeo quando a página é carregada;
  • controls: Customiza os controles a serem mostrados;
  • loop: Quando o vídeo termina, volta para o inicio;
  • muted: Controle para deixar mudo o vídeo;
  • poster: determina a imagem a ser apresentada enquanto o vídeo é carregado ou o usuário clica no play;
  • preload: Carregamento do vídeo quando é lida a página(auto metadata none).

Utilizando a tag <video>:

                    
                        <video width="800" height="600" controls>
                            <source src="movie.mp4" type="video/mp4">
                            <source src="movie.ogg" type="video/ogg">
                            Seu navegador não suporta este recurso.
                        </video>

No caso do canvas, HTML5 nunca, em momento algum, fez qualquer animação. Isso porque o HTML5 é uma simples linguagem de marcação. Dessa forma, o que é necessário para criar animações com HTML5? O que são APIs do HTML5?

O HTML 5, como linguagem de marcação somente, não faz verão. Por isso para aproveitar as funcionalidade de algumas API, usamos o javascript para manipular as propriedades do DOM - Document Object Model - e do BOM - Browser Object Model.

Para criar animações usamos a API <canvas>, que permite junto a linguagem de programação javascript fazer animações.

APIs HTML são funcionalidade criadas para que a criação de websites com muitos recursos a serem usados pelos desenvolvedores através de javascript, typescript e outras linguagens de front-end.

Podemos listar APIs HTML5 como, por exemplo: Validação de formulários, Controles de áudio e vídeo, Canvas, Geolocalização, Aramzenamento local, Caches, Históricos, favoritos e etc...

Como o CSS3 pode ser aplicado na aparência da multimídia HTML5. Demonstre com um exemplo.

Usando os seletores abaixo podemos mudar a aparência da multimidia:

  • -webkit-media-controls-panel
  • -webkit-media-controls-play-button
  • -webkit-full-page-media::-webkit-media-controls-panel
  • -webkit-media-controls-volume-slider
  • -webkit-media-controls-time-remaining-display
  • -webkit-media-controls-volume-slider-container
  • -webkit-media-controls-mute-button
  • -webkit-media-controls-timeline
  • -webkit-media-controls-current-time-display
  • -webkit-media-controls-timeline-container

                        <video id="video" controls>

                    </div>

CSS (style.css):

                    
                        html, body {
                            height: 100%;
                            width: 100%;
                          }
                          
                          .video-player {
                              display: flex;
                              align-items: center;
                              justify-content: center;
                              height: 100vh;
                          }
                          
                          video {
                              width: 100%;
                              max-width: 800px;
                          }
                          
                          /* BOTÃO PLAY  */
                          video::-webkit-media-controls-play-button {
                              background-color: #083147;
                              margin: 0 0.5rem 0.5rem 0;
                              height: 1.5rem;
                              border-radius: 5px;
                          }
                          
                          /* ALTERA VOLUME  */
                          video::-webkit-media-controls-volume-slider {
                            background-color: #083147;
                            padding-top: 0;
                            margin-top: 20px;
                            padding-bottom: 0;
                          }
                          
                          /* ALTERA VOLUME  */
                          video::-webkit-media-controls-mute-button:hover {
                              background-color: #083147;
                              border-radius: 50%;
                          }
                          
                          /* ALTERA TIME LINE  */
                          video::-webkit-media-controls-timeline {
                            background-color: #083147;
                            padding-bottom: 0;
                            margin-bottom: 20px;
